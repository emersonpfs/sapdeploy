### Target Server Requirements

**Linux Servers:**
- SSH access enabled
- User with sudo privileges
- Port 22 open

**Windows Servers:**
- WinRM enabled (HTTP: 5985 or HTTPS: 5986)
- User with Administrator privileges
- Firewall configured for WinRM

1. **Change Default Passwords** - Never use sample passwords
2. **Use SSH Keys** - Prefer key-based auth for Linux

4. **Secure Encryption Key** - Keep .env file protected
5. **Network Segmentation** - Run on isolated management network

## ğŸ› Common Issues

### SSH Connection Fails
```bash
# Test manual SSH first
ssh -i /path/to/key user@server

# Fix key permissions
chmod 600 /path/to/key
```

### WinRM Connection Fails
```powershell
# On Windows server, run as Administrator:
winrm quickconfig
winrm set winrm/config/service/auth '@{Basic="true"}'
```

### WebSocket Connection Issues
- Check if backend is running
- Verify proxy configuration (if using nginx)
- Check browser console for errors
